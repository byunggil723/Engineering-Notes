# 크루스칼

### 시간 복잡도

$$
\text{주어진 간선의 개수가 E일 때},
$$

$$
O(Elog{E})
$$

### 개요

주어진 무방향 연결 그래프에서 MST(최소 신장 트리, Minimum Spanning Tree)를 찾을 때 사용하는 알고리즘.

MST는 무방향 연결 그래프 위에서 정의되며, 모든 노드를 최소 비용으로 연결하는 트리를 말한다.<br>
만약 주어진 그래프가 비연결 그래프인 경우에는 전체를 하나의 트리로 만들 수 없으므로, 각 연결 요소별 MST를 구한 MSF(Minimum Spanning Forest)를 찾아야 하고,<br>
방향 그래프인 경우에는 그래프가 연결되어 있더라도 시작 정점에 따라 도달할 수 없는 정점이 존재할 수 있기 때문에<br>
별도의 알고리즘을 통해 MSA(Minimum Spanning Arborescence)를 찾아야 한다.

MST를 찾기 위해 사용되는 알고리즘으로는 대표적으로 Kruskal 알고리즘과 [Prim](/Algorithm/Prim.md) 알고리즘이 있다.<br>
두 알고리즘 모두 간선의 가중치에 음수가 존재하더라도 사용 가능하며, 결과적으로 모든 정점을 최소 비용으로 연결하는 것이 목적이므로<br>
시작 정점의 선택은 중요하지 않다.(단, 선택되는 MST의 형태는 달라질 수 있다.)<br>
다만 Kruskal 알고리즘은 비연결 그래프가 주어지더라도 각 연결 요소에 대해 MST를 구성함으로써 MSF를 구할 수 있는 반면,<br>
Prim 알고리즘은 연결 그래프에서 MST만 구할 수 있다.

Kruskal 알고리즘은 먼저 주어진 모든 간선을 가중치 기준으로 오름차순 정렬 후, 정렬된 간선 목록에서 가중치가 가장 작은 간선부터 하나씩 선택한다.<br>
이후 유니온-파인드 알고리즘을 통해 현재 선택된 간선이 이미 MST에 속해 있는지 여부를 판단하여,<br>
속해 있지 않는 경우에만 해당 간선을 MST에 포함시키고 전체 비용을 갱신한다.
