# 동적 계획법(Dynamic programming)

### 개요

DP는 주어진 문제가 동일한 구조를 갖는 더 작은 부분 문제들의 반복으로 표현될 때,<br>
각 부분 문제 내에서는 가능한 선택을 모두 고려해 최적값만 남겨 두고, 이를 바탕으로 더 큰 문제의 최적해를 재귀적으로 빠르게 구성해 나가는 기법이다.<br>
이를 위해서는 각 부분 문제에서 최적값을 정의할 기준이 되고, 문제 해결 과정에서 진행 단계를 표현할 수 있는 적절한 상태 설정이 필요하다.<br>

DP를 적용하기에 적절한 문제인지를 판단하기 위해서는, 현재의 선택이 미래의 결과에 영향을 미치는지를 살펴보아야 한다.<br>
그러나 처음부터 하나씩 현재 당장 불리한 선택을 가정해 보고, 그 선택이 오히려 전체적으로 적합한 선택이 되도록 만드는 입력을 직접 구성하는 것은 매우 어렵다.

따라서 이러한 방식 대신, 먼저 주어진 문제를 DP로 해결한다고 가정해 보자.<br>
그리디 알고리즘을 사용한다고 가정하면, 문제 해결 과정에서 항상 하나의 상태만을 유지하게 되므로, 서로 다른 상태가 동시에 공존하는 상황 자체를 고려할 수 없게 된다.<br>
반면 DP를 가정하면, 모든 가능한 상태에 대해 최적값을 기록하므로, 어느 시점에 누적값은 같지만 상태만 서로 다른 경우가 충분히 존재할 수 있다.<br>
이때 이후에 주어지는 입력에 따라 이 두 상태의 결과가 서로 달라진다면, 이는 그다음의 해가 현재 상태에 의존한다는 것을 의미한다.

즉, 현재까지의 누적값만으로 이후의 최적해를 판단할 수 없으므로,<br>
하나의 상태만을 유지하는 그리디 방식이 아닌, 모든 상황을 최적으로 고려하는 DP로 문제를 해결하는 것이 타당하다는 결론을 내릴 수 있다.
