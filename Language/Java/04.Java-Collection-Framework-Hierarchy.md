# Collection Framework 상속 구조

### Collection 계열

```
[인터페이스]
Iterable<E>
    └── Collection<E>
             ├── List<E>
             ├── Set<E>
             │    └── SortedSet<E>
             │            └── NavigableSet<E>
             └── Queue<E>
                    └── Deque<E>


[추상/구현 클래스]
AbstractCollection<E>  → 추상 클래스
         ├── AbstractList<E>  → 추상 클래스
         │        ├── ArrayList<E>  → 구현 클래스
         │        ├── Vector<E>  → 구현 클래스
         │        │      └── Stack<E>  → 구현 클래스
         │        └── AbstractSequentialList<E>  → 추상 클래스
         │                      └── LinkedList<E>  → 구현 클래스
         ├── AbstractSet<E>  → 추상 클래스
         │        ├── HashSet<E>  → 구현 클래스
         │        │       └── LinkedHashSet<E>  → 구현 클래스
         │        └── TreeSet<E>  → 구현 클래스
         ├── AbstractQueue<E>  → 추상 클래스
         │          └── PriorityQueue<E>  → 구현 클래스
         └── ArrayDeque<E>  → 구현 클래스


[각 클래스가 구현하는 인터페이스]
AbstractCollection<E> implements Collection<E>

AbstractList<E> extends AbstractCollection<E> implements List<E>
AbstractSet<E> extends AbstractCollection<E> implements Set<E>
AbstractQueue<E> extends AbstractCollection<E> implements Queue<E>
ArrayDeque<E> extends AbstractCollection<E> implements Deque<E>

ArrayList<E> extends AbstractList<E> implements List<E>
Vector<E> extends AbstractList<E> implements List<E>
Stack<E> extends Vector<E>

AbstractSequentialList<E> extends AbstractList<E>
LinkedList<E> extends AbstractSequentialList<E> implements List<E>, Deque<E> // 따라서 Queue도 포함

HashSet<E> extends AbstractSet<E> implements Set<E>
LinkedHashSet<E> extends HashSet<E> implements SequencedSet<E>
TreeSet<E> extends AbstractSet<E> implements NavigableSet<E> // 따라서 Set, SortedSet도 포함

PriorityQueue<E> extends AbstractQueue<E> implements Queue<E>
```

### Map 계열

```
[인터페이스]
Map<K, V>
 └── SequencedMap<K, V>
          └── SortedMap<K, V>
                  └── NavigableMap<K, V>


[추상/구현 클래스]
AbstractMap<K, V>  → 추상 클래스
      ├── HashMap<K, V>  → 구현 클래스
      │      └── LinkedHashMap<K, V>  → 구현 클래스
      ├── TreeMap<K, V>  → 구현 클래스
      ├── WeakHashMap<K, V>  → 구현 클래스
      └── Hashtable<K, V>  → 구현 클래스


[각 클래스가 구현하는 인터페이스]
AbstractMap<K, V> implements Map<K, V>

HashMap<K, V> extends AbstractMap<K, V> implements Map<K, V>
LinkedHashMap<K,V> extends HashMap<K,V> implements SequencedMap<K,V>

TreeMap<K,V> extends AbstractMap<K,V> implements NavigableMap<K,V>

Hashtable<K, V> extends Dictionary<K, V> implements Map<K, V>

WeakHashMap<K, V> extends AbstractMap<K, V> implements Map<K, V>
```
