# 접근 제어자 (Access Modifier)

## 패키지

패키지는 선언으로 정해지는 논리적 소속을 말한다.<br>
파일 `src/component/A`과 파일 `src/util/B` 상단에 동일하게 `package main_page`가 선언되어 있다면,<br>
개념적으로는 같은 패키지에 속하므로 접근 제어에서 **같은 패키지 기준**이 적용된다.<br>
다만 Java에서는 보통 패키지 선언과 디렉토리 구조를 일치시켜 사용한다.
별도의 패키지 선언이 없으면 해당 클래스는 `default package`에 속한다.

## import

`import`는 `패키지명 + 클래스명`을 써서 다른 패키지의 클래스를 현재 파일에서 사용할 수 있도록 하는 선언이다.

```java
import java.util.Scanner;

// java.util 패키지의 Scanner 클래스를 가져오는 것
```

다른 패키지에 속한 클래스를 사용하려면 보통 `import`가 필요하지만, `import` 없이도 `패키지명.클래스명` 형태로 전체 이름을 직접 명시하여 사용할 수 있다.<br>
같은 패키지에 속한 클래스들끼리는 별도의 `import` 없이 서로의 클래스를 사용할 수 있다.<br>

## 멤버 접근 제어

| 제어자       | 같은 클래스 | 같은 패키지 | 다른 패키지 자식 클래스 | 다른 패키지 일반 클래스 |
| --------- | ------ | ------ | ------------- | ------------- |
| public    | O      | O      | O             | O             |
| protected | O      | O      | O             | X             |
| default   | O      | O      | X             | X             |
| private   | O      | X      | X             | X             |

\* `protected`는 기본적으로 같은 패키지까지 허용하되, 예외적으로 다른 패키지의 자식 클래스에서도 접근을 허용하는 제어자라고 생각하면 된다.

\* `private`은 자식 클래스에서도 접근할 수 없다.

### static

`static`은 멤버를 객체가 아니라 클래스 자체에 귀속시키는 일반 제어자이다.<br>
즉, static으로 선언된 멤버는 객체마다 생성되는 것이 아니라<br>
클래스 단위로 하나만 존재하여 프로그램 실행 동안 공유된다.<br>

따라서, 다음과 같은 프로그래머스의 실행 환경에서는 이전 테스트의 값이 누적될 수 있으니 주의해서 사용해야 한다.

```Java
new Solution().solution(test1);
new Solution().solution(test2);
new Solution().solution(test3);
```

또한 `static` 메서드는 객체 없이도 호출될 수 있는 메서드이므로,<br>
`static` 멤버에는 직접 접근할 수 있지만, 인스턴스 멤버에는 객체를 통해서만 접근할 수 있다.

```java
class A {
  static int s = 10;
  int x = 20;

  static void f() {
    System.out.println(s); // 가능
    System.out.println(x); // 불가

    A a = new A();
    System.out.println(a.x); // 가능
  }
}
```

## 클래스 접근 제어

| 제어자       | 같은 패키지 | 다른 패키지 |
| --------- | ------ | ------ |
| `public`  | O      | O      |
| `default` | O      | X      |

`public class A`<br>
→ 어떤 패키지에서든 A를 사용할 수 있다. (클래스 이름을 다른 곳에서 참조할 수 있다.)<br>
`class A (default)`<br>
→ 같은 패키지에서만 A를 사용할 수 있다.<br>
