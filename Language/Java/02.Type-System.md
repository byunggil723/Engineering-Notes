# Type System

|  primitive 	|  wrapper  |       size      |          scope            |
|-------------|-----------|-----------------|---------------------------|
| byte	      | Byte      | 1바이트 (8비트)  | $-128 \sim 127$           |
| short       | Short     | 2바이트 (16비트) | $-32{,}768 \sim 32{,}767$ |
| int	        | Integer   | 4바이트 (32비트) | $-2^{31} \sim 2^{31} - 1$ |
| long	      | Long      | 8바이트 (64비트) | $-2^{63} \sim 2^{63} - 1$ |
| float	      | Float     | 4바이트 (32비트) | $\pm 1.4 \times 10^{-45} \sim \pm 3.4 \times 10^{38}$ |
| double	    | Double    | 8바이트 (64비트) | $\pm 4.9 \times 10^{-324} \sim \pm 1.8 \times 10^{308}$ |
| char	      | Character | 2바이트 (16비트) | $0 \sim 65{,}535$         |
| boolean	    | Boolean   | -               | -                         |

\* `char`는 문자를 UTF-16으로 표현했을 때의 코드 유닛 1개를 담는다.<br>
이때 어떤 문자가 UTF-16 코드 유닛 1개로 표현되지 않는 경우, 하나의 char에 담을 수 없고 **두 개의 char**를 이용해 표현한다.<br>
이런 문자는 보통 `String` 안에서 다루게 된다.

## 1. Primitive

### 1\) 리터럴 타입

```java
// 정수
10      // 기본적으로 int
10L     // long
10l     // long (되긴 하지만 l은 1이랑 헷갈려서 잘 안 씀)

// 실수
3.14    // 기본적으로 double
3.14f   // float
3.14F   // float
3.14d   // double
3.14D   // double

// 진법, 아래는 모두 10진수 10을 의미
10      // 10진수
012     // 8진수, int x = 019; -> 에러
0xA     // 16진수
0b1010  // 2진수

// 언더 스코어(구분자)
int a = 1000000;
int b = 1_000_000;
```

### 2\) 대입 규칙

자바에서는 기본적으로 확대 변환이 가능하므로, 표현 범위가 더 작은 타입의 값을 표현 범위가 더 큰 타입의 변수에 대입할 수 있다.<br>
반대로, 표현 범위가 더 큰 타입의 값을 더 작은 타입의 변수에 대입하는 것은 자동으로 허용되지 않으며, 이러한 경우에는 캐스팅(명시적 형변환) 이 필요하다.<br> 
이때는 값의 손실이 발생할 수 있다.<br>

```
# 명시적 형변환 없이 자동으로 대입 가능한 순서
byte -> short -> int -> long -> float -> double
char -> int -> long -> float -> double
```

```java
public class Main {
  public static void main(String[] args) {
    double a = 3.141592653589793;

    float b = 3.14;       // 에러
    float b = (float) a;  // 가능

    System.out.println("double a = " + a);  // 3.141592653589793
    System.out.println("float  b = " + b);  // 3.1415927

    char c = 65;        // 가능
    short s = 100;      // 가능

    short x = 65;
    char y = x;         // 에러
    char z = (char) x;  // 가능
  }
}
```

### 3\) 연산 규칙

`primitive` 타입 간의 산술 연산(+, -, *, /, %)에 대해서는 다음의 규칙이 적용된다.

```
# 산술 연산 시 자동으로 다음의 순서로 타입이 승격됨
byte, short, char -> int -> long -> float -> double

int + long -> long
int + double -> double
long + double -> double
float + double -> double
```

```java
byte a = 10;
byte b = 20;

int c = a + b;   // 가능
byte d = a + b;  // 에러

int x = 2000000000;
int y = 2000000000;
long z = x + y; // long z = (long)x + y; 와 동일
```

\* String의 경우, `+` 연산에서만 특별히 문자열 결합 규칙이 적용된다.

```java
System.out.println(1 + 2 + "3");  // 33
System.out.println("1" + 2 + 3);  // 123
```

## 2. Wrapper Class

List, Set, Map 등의 자료구조를 통틀어 컬렉션(Collection Framework)이라고 부른다.<br>
컬렉션은 객체(Object)만 저장할 수 있어, 반드시 Wrapper Class(Integer, Long 등)를 사용하여 선언한다.

```Java
ArrayList<int> list;        // 불가능
ArrayList<Integer> list;    // 가능
```

### 1\) 대입/연산/캐스팅

wrapper class끼리는 대입/캐스팅이 불가능하며, 연산은 unboxing 후 primitive 규칙이 적용된다.

### 2\) 오토박싱, 언박싱 (Autoboxing, Unboxing)

primitive 타입과 Wrapper Class 사이에는 자동 변환이 발생한다.

#### primitive → 객체

```Java
int a = 10;
Integer b = a;   // 오토박싱: 내부적으로 Integer b = Integer.valueOf(10); 를 수행
```

#### 객체 → primitive

```Java
Integer a = 10;
int b = a;   // 언박싱: 내부적으로 int b = a.intValue(); 를 수행
```

### 3\) 비교 연산자

primitive 타입은 JVM 메모리 상에서 변수에 값 자체가 저장되고,<br>
reference 타입은 변수에 해당 객체를 가리키는 참조(실제 물리 주소가 아닌, JVM이 관리하는 추상적 주소)가 저장된다.<br>
따라서 primitive 타입은 값 자체를 저장하므로 `==`로 바로 값의 비교가 가능하다.<br>
반면, Wrapper Class와 같은 reference 타입은 변수에 객체의 주소가 저장되므로 `==`는 주소 비교를 수행한다.<br>
따라서 값 자체의 비교를 위해서는 `equals()` 메서드를 사용해야 한다.

```Java
Integer a = new Integer(10);
Integer b = new Integer(10);

a == b        // false (서로 다른 객체)
a.equals(b)   // true  (값 자체 비교)
```

Integer와 같은 일부 Wrapper 클래스는 -128 ~ 127 범위의 값에 대해 캐싱이 적용되어, 동일한 값이 할당될 경우 내부적으로 동일한 객체가 재사용된다.<br>
따라서 해당 범위 내에서는 == 비교 연산이 값 자체를 비교하는 것처럼 동작한다.<br>
위의 예시에서는 비교 대상 값이 해당 범위 내에 있지만, `new`를 통해 각각 새로운 객체로 생성되었기 때문에<br>
`==` 비교 시 서로 다른 객체로 판단되어 `false`가 된다.

```Java
Integer a = 100;
Integer b = 100;

a == b        // true
a.equals(b)   // true
```

하지만, 이러한 별개의 동작에 의존하지 말고, 객체 간 값 비교는 항상 `equals()` 메서드를 사용하는 것이 안전하다.<br>
단, primitive 타입(int, long 등)은 객체가 아니므로 `equals()`를 사용할 수 없으며, 이 경우에는 반드시 `==`를 사용하여 값 비교를 수행한다.

String 리터럴은 String pool에 저장되며, 이 또한 동일한 문자열이 할당될 경우 내부적으로 동일한 객체가 재사용된다.

```Java
String a = "abc";
String b = "abc";

a == b        // true  (같은 객체)
a.equals(b)   // true
```

그러나 `new`를 사용하면 역시 새로운 객체가 생성된다.

```Java
String a = new String("abc");
String b = new String("abc");

a == b        // false
a.equals(b)   // true
```
