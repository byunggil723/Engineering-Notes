# 리액트 생명 주기

1. useState가 관리하는 상태 변화에 따른 리렌더링은 useState가 선언된 컴포넌트를 기점으로 일어 난다.<br>
   예를 들어, A 컴포넌트에 useState가 선언되고, B 컴포넌트에는 setState 함수가, C 컴포넌트에는 state가 각각 props로 전달되었을 때, state의 변화가 일어나면 A부터 먼저 리렌더링이 시작된다. 이때 A 컴포넌트와 C 컴포넌트 내부에 useEffect가 선언되어 있다면, C 컴포넌트의 useEffect 부터 평가된다.<br>
   즉,<br>
   리렌더링 순서: A -> C<br>
   useEffect 반영 순서: C -> A

2. 커스텀 훅 useCustom이 A 컴포넌트 내부에 선언되어 있다 하자.<br>
   커스텀 훅 내부에 useState가 선언되어 있고, 어떠한 조건에 의해 이 state가 변하면 커스텀 훅을이 선언되어 있는 A 컴포넌트 전체가 리렌더링된다. 이때, 이 렌더링에는 이미 변경된 state값이 반영되어 있다.

3. 한 번의 렌더 사이클 내에서 실행되는 모든 setState에 의한 상태 변화는 묶어서 그 다음 렌더 사이클에 일괄적으로 반영한다.

4. 렌더링 중, setState에 의해 변경된 상태는 내부적으로 관리가 되고 있다.<br>
   이때 만약 직접 대입형으로 setState를 사용한다면, 형태 상 내부적으로 관리되고 있는 상태를 참조할 수 없기 때문에 다음과 같이 동일한 setState의 함수를 한 번의 렌더 사이클 내에 여러 번 사용하면 마지막 setState만 반영이 된다.<br>
   따라서 렌더링 중 내부적으로 관리되는 값을 참조하기 위해서는 setState를 함수형 업데이트 방식으로 사용해야 한다.
